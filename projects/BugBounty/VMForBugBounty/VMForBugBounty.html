Setting up a VM for Bug Bounty

Creating the VM

  1. Navigate to the Kali Linux website and locate the cloud installation section.
  2. Click the marketplace button under Microsoft Azure. This will take you to the Azure Marketplace where we can buy Kali Linux on Azure. Click the “Get It Now” button.
  3. This will take you to the Kali Linux page in your Azure portal. Click the “Create” button to configure the VM.
  4. I created a new resource group called Kali_Linux_BugBounty. I gave the VM a name and used an existing key stored on Azure.
  5. I created a new virtual network called kali-network.
  6. I kept the rest of the settings as the default. When you’re done configuring the VM click the “Review + Create” button.
  7. Once you have reviewed the configuration and everything is correct click the “Create” button.

Creating Remote Connection

  1. In the Azure portal navigate to the kali-linux machine then navigate to the connect section.
  2. Open a terminal in a Linux machine. I’m using WSL.
  3. Select the “Native SSH” option and enter the location of the SSH key on the local Linux machine. This will give you the command you need to run to connect to the machine.
  4. Run the command as root to start the connection.

Creating a Backup of the VM

  1. In the Azure portal navigate to the kali-linux machine.
  2. Click the “Capture” button then click “Restore Point”.
  3. Give the restore point a name and create a restore point collection to store it in.
  4. When you’re done configuring the restore point click the “Review + Create” button.
  5. Once you have reviewed the configuration and everything is correct click the “Create” button.
