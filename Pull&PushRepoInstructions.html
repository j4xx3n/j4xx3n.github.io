How to Push Changes to the Main Branch

1. Check the Status of Your Changes

Run the following command to check the status of your changes:

git status

This will show you the files that have been modified or added.

2. Stage Your Changes

Add the files you want to include in the commit:

git add <file_name>

To stage all changes at once:

git add .

3. Commit Your Changes

Add a meaningful commit message:

git commit -m "Your commit message here"

4. Push Your Changes to the Main Branch

Assuming your main branch is named main (verify with git branch):

git push origin main

How to Update a Repository After Cloning

1. Check for Updates

Run the following command to fetch the latest changes from the remote repository:

git fetch

This will retrieve updates without applying them to your local branch.

2. Merge Updates

To update your current branch with the fetched changes, run:

git merge origin/main

This assumes your main branch is named main.

3. Pull Changes Directly

Alternatively, you can fetch and merge in one step using:

git pull origin main

4. Resolve Conflicts (if any)

If there are merge conflicts, Git will notify you. Open the conflicting files, resolve the issues, and then run:

git add <conflicted_file>
git commit

5. Verify Updates

Finally, check the status of your branch to ensure it is up to date:

git status
