1. Cloning the Repo

How to Generate a New SSH Key

1. Open Git Bash

Start by opening Git Bash on your local machine.

2. Generate a New SSH Key

Run the following command, replacing your_email@example.com with your GitHub email address:

ssh-keygen -t ed25519 -C "your_email@example.com"

Note: If your system does not support the Ed25519 algorithm, use:

ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

3. Save the Key

When prompted to "Enter a file in which to save the key", press Enter to accept the default file location, or specify a custom location by providing a filename. For example:

Enter file in which to save the key (/c/Users/YOU/.ssh/id_ALGORITHM): [Press Enter]

4. Add a Passphrase

Type a secure passphrase when prompted. If you do not want a passphrase, press Enter without typing anything.


    Enter passphrase (empty for no passphrase): [Type a passphrase]
    Enter same passphrase again: [Type passphrase again]

5. Notes on Key Types

  • GitHub no longer supports DSA keys (ssh-dss).
  • RSA keys generated after November 2, 2021, must use a SHA-2 signature algorithm.
  • Ensure your Git client is up to date for compatibility with modern key algorithms.

How to Add an SSH Key to GitHub

1. Navigate to GitHub Settings

Log in to your GitHub account and navigate to Settings. Then go to the SSH and GPG keys section.

2. Add a New SSH Key

Click on New SSH key.

3. Enter Key Details

Fill in the required fields:

  • Title: Provide a descriptive name for the device connecting to GitHub.
  • Key type: Select Authentication Key.
  • Key: Paste the public key generated on your machine.

4. Save the Key

Click Add SSH key to save your key.

2. Makeing changes to the repo

Before making changes to the repo make sure to update to main branch.

How to Update a Repository After Cloning

1. Check for Updates

Run the following command to fetch the latest changes from the remote repository:

git fetch

This will retrieve updates without applying them to your local branch.

2. Merge Updates

To update your current branch with the fetched changes, run:

git merge origin/main

This assumes your main branch is named main.

3. Pull Changes Directly

Alternatively, you can fetch and merge in one step using:

git pull origin main

4. Resolve Conflicts (if any)

If there are merge conflicts, Git will notify you. Open the conflicting files, resolve the issues, and then run:

git add <conflicted_file>
git commit

5. Verify Updates

Finally, check the status of your branch to ensure it is up to date:

git status

How to Push Changes to the Main Branch

1. Check the Status of Your Changes

Run the following command to check the status of your changes:

git status

This will show you the files that have been modified or added.

2. Stage Your Changes

Add the files you want to include in the commit:

git add <file_name>

To stage all changes at once:

git add .

3. Commit Your Changes

Add a meaningful commit message:

git commit -m "Your commit message here"

4. Push Your Changes to the Main Branch

Assuming your main branch is named main (verify with git branch):

git push origin main
